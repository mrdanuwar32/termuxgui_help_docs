Help on package termuxgui:

NAME
    termuxgui - This is a library for interacting with the Termux:GUI plugin from python.

DESCRIPTION
    You don't need to include all the submodules, all needed classes are automatically included in the package scope upon
    importing the package.

    class Buffer(builtins.object)
     |  Buffer(connection, w: int, h: int, format: Literal['ARGB888'] = 'ARGB888')
     |
     |  This represents a raw ARGB888 image buffer.
     |
     |  This is only available on Android 8.1+.
     |  Buffers consume much memory, so be sure to close any Buffer you don't need any more.
     |  Preferably use "with" to get the memory object of the buffer so it is automatically closed.
     |  The memory object is a shared memory object that can be used like a bytes object or a file object.
     |  You can also directly access the memory object as b.mem.
     |
     |  A RuntimeError is raised when a Buffer can't be created.

    class Connection(builtins.object)
     |  This represents a connection to the Termux:GUI plugin and contains all functions that don't act on any
     |  particular View, Activity or Task.
     |
     |  Methods defined here:
     |
     |  __enter__(self)
     |
     |  __exit__(self, type, value, traceback)
     |
     |  __init__(self)
     |      When a connection can't be established, a RuntimeError is raised.
     |
     |  checkevent(self) -> Optional[termuxgui.event.Event]
     |      If there is at least one event to be read, returns it.
     |      If there is no event, returns None.
     |      You can use this to e.g. check for events between drawing a frame instead of
     |      having to use a separate thread and blocking it.
     |
     |  close(self)
     |      Closes the connection.
     |
     |      The connection is automatically closed when the program exits, but it's good practice to close the connection
     |      yourself when you don't need it anymore or use a "with" statement.
     |
     |  events(self) -> Iterator[termuxgui.event.Event]
     |      Waits for events. Use this with "for in" to iterate over incoming events and block while waiting.
     |
